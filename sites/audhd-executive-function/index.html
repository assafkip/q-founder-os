The problem

AUDHD founders face compounding executive function challenges that generic productivity tools ignore entirely.

Working memory

If a task, person, or commitment is not visible on screen right now, it does not exist. Both ADHD and ASD compromise working memory independently. Combined, the deficit compounds.

Wall of Awful

Emotional barriers built from accumulated failure. Blank-page paralysis on every outreach email, every follow-up, every ask. Not laziness. Self-protection.

Decision fatigue

ADHD brains deplete dopamine faster during decision-making. Too many choices triggers freeze. "Who do I contact first? What do I say?" becomes paralysis.

RSD + outreach

Rejection Sensitive Dysphoria makes every cold message feel like a personal risk. The most common coping mechanism: refusing to try at all.

What the skill does

Every feature exists to eliminate a specific executive function gap. Nothing decorative.

Friction-sorted ordering

Tasks ordered by friction, lowest first. 2-minute replies before 5-minute DMs before 10-minute emails. Momentum builds before harder tasks surface.

Copy-paste everything

Every item has the actual message text with a Copy button. Emails include subject lines. DMs are ready to paste. The compose step is eliminated entirely.

No shame language ever

No "overdue." No "missed." No "failed." Carried forward. Ready when you are. Awaiting reply. Effort tracked, never outcomes. RSD-safe by default.

Zero-decision execution

The system picks who to contact, what to say, in what order, through which channel. You copy, paste, and check off. Override if you want, but you never have to decide.

Energy-tagged tasks

Every item tagged: Quick Win, Deep Focus, People, or Admin. Similar-energy tasks batched together. Context-switching minimized. Transition buffers included.

Automatic crack detection

Surfaces what fell through without you remembering to check. Awaiting reply over 7 days. Meeting follow-ups not sent. Cooling contacts. All with recovery drafts attached.

Inline relationship context

Every person-related task shows last interaction, what was discussed, and what you owe them. Context and action in one place. Never search your memory.

Skip without shame

Missed items move to "carried forward" without commentary. Every section is independently completable. Missing one module does not invalidate others.

What this skill never does

These are hard rules. The skill actively blocks these patterns from appearing in any output.

Dashboards without actions

A score means nothing unless the recovery draft is right there. Pipeline counts without per-person actions are analytics, not executive function.

Options to evaluate

"Here are 3 comment styles, which do you prefer?" is a decision. The system picks. One comment, ready to copy. Override if you want.

Abstract task names

"Follow up with Jay" is not a task. The actual email text is the task. Every item contains the literal next physical action.

Shame language

No "overdue," "missed," "failed," "forgot," "dropped the ball," "behind." Effort tracked, never outcomes. RSD protection is structural.

Cross-references

"See section above" or "copy from file X" is a working memory tax. Everything is inline. The user never scrolls or searches.

Outcome metrics

"0 replies received" triggers RSD. Track effort only. "You sent 4 messages yesterday" is the metric. What came back is not your score.

Built for AUDHD brains

Design decisions grounded in AUDHD research. Barkley's scaffolding framework, Mahan's Wall of Awful, Dodson's interest-based nervous system model.

01
The system IS the executive function

Based on Barkley's scaffolding: externalize working memory, planning, and initiation into the environment. The daily HTML is not a briefing. It is the user's entire workday, pre-loaded.

02
Momentum-first, not eat-the-frog

The interest-based nervous system needs activation energy. Start with quick wins to build completion dopamine. Then surface harder tasks while momentum is flowing. Never start with the hardest item.

03
Flexible rails, not rigid routines

AUDHD paradox: craves predictable routines AND rebels against monotony. Solution: consistent structure with dynamic content. Layout never changes. Each section independently completable. Skipping one never invalidates others.

04
Sensory-safe visual design

Muted dark palette. No animations or flashing. High information density, low visual noise. Typography hierarchy (size, weight) for importance, not color. Single column. Works on phone.

05
Progress visibility as dopamine

Completion is a neurological accommodation. Visual progress bar at top. Completed items stay visible (struck through). End-of-day effort summary. Show what WAS done, never what wasn't.

06
Body doubling through persistent presence

The dashboard is always present. It knows what you committed to. Shows what you are working on. Witnesses without judging. A digital body double that never sighs, checks the clock, or looks disappointed.
